## Description
Fixes Firebase Auth Token Refresh 400 Bad Request errors often encountered during local emulator development. This occurs when emulator restarts wipe data while the app still holds a stale session, or due to time skew.

This PR implements:
- Graceful error handling in `AuthContext` to intercept `400 Bad Request` or `network-request-failed` errors on token refresh.
- Auto-recovery: Automatically attempts a silent re-authentication using the user's saved credentials securely stored on the device.
- Fallback sign-out: If auto-recovery fails, signs the user out cleanly rather than leaving the app in an unrecoverable state.
- Documentation updates in `FIREBASE_SETUP.md` explaining these emulator token refresh issues and manual workarounds.
